      GIT Branching

      The default branch name in GIT is 'master' . As oyu start making commits you are given a master and it points to the last commit .Every time you commit, the master branch pointer moves forward automatically.
    12. git branch git branch
    13. git current branch location $ git log --oneline --decorate

      14. The branch hasn’t disappeared; Git just doesn’t know that you’re interested in that branch and it is trying to show you what it thinks you’re interested in. In other words, by default, git log will only show commit history below the branch you’ve checked out.

    14. git show all loggit log --all
    15. git show log to specific branch git log branch-name
    16. Tip - checking out and creating a new branch at the same time git checkout -b new branch-name
    17. from 2.3 version NEW Command 'switch' git switch -c new branch-name
    18. git merge branch-name
    19. deleting a branch git branch -d branch-name

      20. MERGING BASICS

    20. if you want to merge your new changes with master or other branches FIRST check out the branch you wish to merge into git checkout master-or-your-desired-branch and SECOND run the git merge command git merge branch-that-you-want-to-fix like git merge iss53
      21. Instead of just moving the branch pointer forward, Git creates a new snapshot that results from this three-way merge and automatically creates a new commit that points to it. This is referred to as a merge commit, and is special in that it has more than one parent.
